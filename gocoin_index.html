Core Components

Client

Full node implementation with network communication and memory pool. HTTP-based interface for remote access.

  • Up-to-date consensus rules
  • Web-based interface
  • Mining capable

Wallet

Deterministic and password seeded. Designed for offline use. BIP-39 compatible with popular wallets.

  • Offline operation
  • No backups needed
  • Independent from client

Tools

Additional utilities included in the tools/ directory. Each subdirectory contains a separate tool.

  • Command-line utilities
  • Blockchain analysis
  • Transaction tools

Architecture

Unlike Bitcoin Core, the full node client is an application independent from the wallet. This separation allows for flexible deployment and offline wallet security.
